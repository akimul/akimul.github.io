Git subtree

Using git subtree

 Posted on February 21, 2019  |  2 minutes  |  359 words  |  Akimul Islam Akash

Overview:

Suppose, we have repository. We want to share another repository as a library between multiple repositories. In that case, we subtree the shared repository in other repositories.

Usage and Commands:

Let say, we have two repositoty parent and child. We want to use the child repo as subtree library in the parent repo.

Initially,

parent
--a.py
--b.py

and child

child
--c.py
--d.py

adding subtree

In order to add the child repo as subtree to the parent repo:

git subtree add --prefix child child_repo master --squash

This will fetch the master branch from the child repo and put it in a folder name child defined by the --prefix. this records a merge commit. The common practice is to not store the entire history of the subproject in your main repository, but If you want to preserve it just omit the –squash flag. Now:

parent
--a.py
--b.py
--child
----c.py
----d.py

pushing to subtree

Let us now assume that we have made some changes in d.py or we have created a new file in child named e.py. Then we commit and push those changes using normal git push origin master‘ Everything is okay, changes will be pushed to parent. But it will not be reflected to child repo. In order to push to subtree repo:

git subtree push --prefix child child_git_repo master

You need to run this command from the toplevel of the working tree.

pulling the subtree

As the subtree repo is shared between other repos, let us assume that someone has made some changes on the subtree repo and we want to pull those changes in our parent repo. In parent run:

git subtree pull --prefix child child_git_repo master --squash

If you do git pull child_git_repo master, to get the latest files for that repo you need to do it from subtree(child) dir.

Switching to different subtree branch

If we want to switch to different branch of the subtree, we need to first the the subree(child). From the parent dir:

git rm -r child
git commit -m 'msg'

now we need to add the subtree again with the desired branch name:

git subtree add --prefix child child_repo new_branch --squash
