Configuring Webpack and ES6 in Django

 Posted on February 18, 2019  |  5 minutes  |  1023 words  |  Akimul Islam Akash

In this post, I will show you how to configure Webpack and ES6 in Django application. I guess you already know django and want to configure webpack in your django app, this post will show you step by step guide.

Tech Stack

  1. Python3
  2. Django 1.11
  3. Webpack4
  4. NPM
  5. Javascript (ES6)

Environment setup & Requirements

Before getting started, please make sure that the followings are already installed on your system:

  • Python (>= 3.0.0)
  • virtualenv
  • nodejs
  • npm

Getting started

First you need to create a virtual environment for the project, install django and create a django project. So lets:

virtualenv -p /usr/bin/python3 virtual_env_path/envname
source virtual_env_path/envname/bin/activate
pip install Django==1.11

Here virtual_env_path is the path where you want to create the virtual env. Now change directory where you want to create the django project and do the followings.

django-admin startproject djwebpack
cd djwebpack
python manage.py startapp djwapp

Now add the newly created app name to the settings file under djwebpack:

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'djwapp',
]

Now if you run:

python manage.py runserver

your django project will be running on localhost:8000.

1. Creating view and pages

Now, it’s time to create first page. We will create a simple django template view and update the url accordingly.

In the djwapp app’s view.py:

view.py

from django.shortcuts import render
from django.views.generic import TemplateView


class HomeView(TemplateView):
   template_name = 'home.html'

Now we need to create the home.html. Before that run the following command from djwebpack directory.

mkdir -p djwapp/templates djwapp/static/js djwapp/static/css

The reason is for organizing the templates and other static files. Now create the home.html file in djwapp/templates with the following:

home.html

{% load static %}
<!DOCTYPE html>
<html>
   <head>
      <title>Home</title>
      <link rel="stylesheet" href="./static/css/main.css">
   </head>
   <body>
      <div class="center">
         <h1 class="name">Hi <span id="name"></span></h1>
         <h3 id="counter"></h3>
         <button id='fetch-btn'>Fetch Data</button>
         <hr>
         <div >
            <table id='data-table'>
               <thead>
                     <tr>
                        <th>Name</th>
                        <th>Year</th>
                        <th>Department</th>
                     </tr>
               </thead>
               <tbody id="table-body">

               </tbody>
               
            </table>
         </div>
         <script src="{% static 'bundle.js' %}"></script>
      </div>
   </body>
</html>

Now create two js files in djwapp/static/js named index.js and timer.js. We will bundle these two js files using webpack in a single file. As you can see from the home.html that script src is bundle.js.

index.js

import * as timer from './timer'

const name = document.getElementById('name')
name.innerText = "John Doe"

timer.startTime()

timer.js

const counter = document.getElementById('counter')
export let checkTime = (i)=> {
   if (i < 10) {i = "0" + i}
   return i
}

export let startTime = () =>{
   let today = new Date()
   let h = today.getHours()
   let m = today.getMinutes()
   let s = today.getSeconds()
   m = checkTime(m)
   s = checkTime(s)
   counter.innerHTML = h + ":" + m + ":" + s
   setTimeout(startTime, 500)
}

We have almost everything setup from django side.

2. Webpack setup

Now it’s time to setup and configure webpack. We want to keep the webpack related files and settings separate from django, so create a folder under your at the project root named client. Project structure will be like:

root
--client
--djwebpack
----djwapp
----djwebpack

Under client run:

npm init

npm will ask for some information,enter them and hit Enter and it will generate a package.json file. Then install webpacka and dependencies for the project:

npm i webpack webpack-cli

You can see the package.json file will be updated. Now we need a webpack config file to instruct webpack what to do. Under client run:

touch webpack.config.js

Open the config file and add the following;

webpack.config.js

let webpack = require("webpack");
let path = require("path");

let DIST_DIR = path.resolve(__dirname, "../djwebpack");
let SRC_DIR = path.resolve(__dirname, "../djwebpack");

let config = {
   mode: 'development',
   entry: SRC_DIR + "/djwapp/static/js/index.js",
   output: {
      path: DIST_DIR + "/dist",
      filename: "bundle.js",
   },
   module: {
      rules: [
        {
          test: /\.js$/,
          exclude: /(node_modules|bower_components)/,
          use: {
            loader: 'babel-loader',
            options: {
              presets: ['@babel/preset-env']
            }
          }
        }
      ]
    },
   devtool: 'source-map'
};

module.exports = config;

Here, few things needs to be made clear. entry the source and entry file of the files that will be bundled. output The output property tells webpack where to emit the bundles it creates and how to name these files loaders Normally webpack only knows js and json files. Loaders allow webpack to process other types of files and convert them into valid modules that can be consumed by your application and added to the dependency graph.

If you want to learn more on webpack, follow their official documentation

Installing babel

As we are using ES6 syntax in our index.js and timer.js, which is supported by most modern browser but not by old browsers. In order to support on old browser, we need to add Babel which a Javascript transpiler that compiles ES6 to ES5.

npm install -D babel-loader @babel/core @babel/preset-env

It will install babel loader and update the package.json file. Here we have used -D to indicate that it is a development dependency. Now webpack configuration is done. We need to update the package.json file a bit. Update the script key with the following value:

{
    "build": "webpack",
    "build:local": "webpack --env.development --watch --hot"
}

Now if you run npm run build:local, webpack will start bundling the files mentioned in the entry and output the bundle in the djwebpack/dist directory.

Updating django static dir

We need one more change in the django settings file. Add the static files dirs:

STATICFILES_DIRS = [
    os.path.join(BASE_DIR, "dist"),
]

Also update the urls.py file to add the home view and static settings:

from django.conf import settings
from django.conf.urls.static import static
from django.conf.urls import url
from django.contrib import admin

from djwapp.views import HomeView

urlpatterns = [
    url(r'^admin/', admin.site.urls),
    url(r'', HomeView.as_view(), name="Home"),
] + static(settings.STATIC_URL, document_root=settings.STATIC_ROOT)

if the django server si running you can see the page and also check that it’s loading the bundle.js which is bundled by webpack. You can get the source code for this at my github.

Multiple js entry for webpack

If there are multiple js files for different templates, you can separate entry in the config file:

entry: {
    app: './js/index.js',
    another: './js/another.js'
},
output: {
    path: DIST_DIR + "/dist",
    filename: '[name].bundle.js'
},

The webpack script will now compile two applications: index.js and another.js. It will create two compiled files based on the entry name: app.bundle.js and another.bundle.js. Then from the required template, we can the required js.
